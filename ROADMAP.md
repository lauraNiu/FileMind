### 1.2 ⚠️ 半成品（UI 有 / 后端缺，或反之）

| 项 | 缺什么 | 影响 |
|---|---|---|
| **Settings 中心** | 只有 popover 展示，无法真正修改 | 用户改预算 / 模型只能编辑 .env |
| **扫描后的文件没摘要** | 后端没自动跑 AI 摘要 | Chat 引用扫描文件时质量低 |
| **图谱"清理决策辅助"** | 后端无操作命令，前端也没接入 | 杀手级用法没落地 |

| **首次 Onboarding** | 没有引导，新用户直接进 Dashboard | 学习成本高 |
| **状态条暂停按钮** | UI 有但点击无效 | 误导用户 |
| **侧栏"我的视图"** | 静态数字，无功能 | 误导 |
| **缩略图预览** | 文件详情抽屉只显示图标 | 找图片场景体验差 |

## 2. 核心价值假设（待验证）

**任何 PM 路线图的前提是：你赌的核心假设是什么。我们押了 5 条：**

| # | 假设 | 风险 | 验证方法 |
|---|---|---|---|
| **H1** | 用户愿意把电脑文件元数据交给本地 AI 工具看 | 中 | 内测 50 用户安装率 + 留存 |
| **H2** | "看清"（先索引不操作）能产生足够吸引力让用户留下 | 高 | D7 留存 > 40% |
| **H3** | 中国用户接受云端 GLM-4 处理元数据（不发原文） | 低 | 调研 + 隐私开关使用率 |
| **H4** | 自然语言问文件 比 传统搜索更高频被使用 | 中 | Chat 使用次数 / 周 ≥ 3 |
| **H5** | 关系图谱在"清理决策"场景有杀手级价值 | 高 | 用户实际使用频率 |

> **PM 视角**：H2 和 H5 是最大风险。如果只是"另一个 Spotlight"，留存上不去。**短期内必须把图谱的"清理决策辅助"做出来作为差异化锚点。**

## 3. 优先级框架

### 3.1 评分维度

每个 backlog 项目用 **RICE 简化版** 评分：

```
Score = Reach × Impact × Confidence ÷ Effort

Reach     = 影响多少用户（1-5：少数 → 全部）
Impact    = 单次影响（1-5：边缘 → 核心体验）
Confidence = 我们对"这个 work"的把握（0.5-1.0）
Effort    = 人天估算
```

### 3.2 三档时间窗

- **短期 P0/P1**：2 周内必做 → v0.2
- **中期 P0/P1**：1-3 个月 → v0.5 Beta
- **长期**：3-12 个月 → v1.0+

### 3.3 北极星指标

| 阶段 | 北极星 | 辅助 |
|---|---|---|
| v0.2 | 首次扫描完成率 ≥ 80% | 0-bug 上线 |
| v0.5 | D7 留存 ≥ 40% | Chat WAU ≥ 3 次 / 周 |
| v1.0 | 付费转化 ≥ 5%（freemium） | NPS ≥ 40 |

## 5. 中期 · 1-3 个月（v0.5 Beta）

> **目标**：完成 Phase 1 PRD 剩余 + 启动 Phase 2 核心。开放 100 人内测，跑出留存数据。

### 5.1 必做模块（按 PRD 节奏）

#### 模块 A · 真正的"实时管家"

- **FSEvents 实时监听**：新增 / 删除 / 重命名秒级同步到索引
- **增量摘要队列**：新文件入队，按预算节流跑摘要
- **批量摘要回填**：历史文件后台慢慢补齐
- **预算控制 + 降级**：超出本月预算自动切本地小模型（先 mock）

#### 模块 B · 语义层

- **本地向量嵌入**（bge-m3 通过 ONNX Runtime / candle）
- **向量库**：sqlite-vss 或 LanceDB（评估）
- **混合搜索**（BM25 + 向量）+ RRF 融合
- **"找相似图片 / 文档"** 用法落地

#### 模块 C · 写操作（Phase 2 起步）

> **重头戏，决定产品价值能不能落地**

- **统一的"操作提议"机制**
  - `operations` 表落地（已有 schema）
  - 所有 AI 行为先入 `pending`，用户审阅后才 `applied`
  - 每条操作可一键回滚
- **场景 1**：临时文件归档向导
- **场景 2**：重复文件清理向导（按"占用 / 价值"排序）
- **场景 3**：截图按"年-月"自动归类（首个自动化规则示例）
- **渐进式自动化**：观察用户接受 N 次同类操作 → 弹"要不要自动化？"

#### 模块 D · 项目识别 v2

- 多信号融合（路径 + 内容相似 + 时间 + 引用）
- AI 给项目命名（不再用目录名）
- 用户可拖拽校正归属

### 5.2 关键体验升级

- **完整的设置中心**（替换 v0.2 的最小可用版）
- **审计日志**：每次云端请求 / 写操作的完整明细
- **每周报告**：「过去一周 AI 帮你做了什么 / 索引了多少 / 答了多少问题」
- **快捷键体系**：除 ⌘K 外，⌘1-5 切页，⌘N 新会话等

### 5.3 内测计划

- 招募 100 用户（产品经理 / 设计师 / 研究者各 30+）
- 每周用户访谈 5 个
- 关键指标日报：DAU / 留存 / Chat 使用 / 扫描完成率
- A/B 测试两套 Onboarding 文案

### 5.4 v0.5 验收清单

- [ ] 100 用户内测，D7 留存 ≥ 40%
- [ ] Chat WAU ≥ 3 次/用户·周
- [ ] 扫描 10 万文件 < 1 小时
- [ ] 至少 3 个真实写操作场景上线
- [ ] 零数据丢失事故

## 6. 长期 · 3-12 个月（v1.0 + 平台化）

> **目标**：从"工具"变"操作系统"。完成 Phase 3 + 启动多平台、生态、商业化。

### 6.1 Phase 3 完整收口

- **版本管理**：v1 / v2 / 最终 自动归族 + 版本树可视化
- **重要文件自动快照**（Git-like 但用户无感）
- **关系图谱进阶**：时间轴 + 关系混合视图
- **完整的清理决策中心**（图谱杀手级用法）

### 6.2 平台扩展

| 平台 | 时机 | 优先级 |
|---|---|---|
| **Windows** | v1.0 之后 | 高（市场大） |
| **Linux** | v1.0 之后 | 中（开发者用户） |
| **移动端伴侣 app**（仅查询 + 通知 + 拍照入库） | v1.5 | 中 |
| **Web 查看器**（只读分享一个项目） | v2.0 | 低 |

### 6.3 本地 LLM 兜底

- 集成 Ollama / llama.cpp，支持 Qwen2.5 / GLM-4-9B 本地版
- 隐私敏感用户的 0 元方案
- 弱网 / 离线场景兜底

### 6.4 生态与协作

- **插件系统**：第三方可贡献文件类型解析器、整理规则、AI prompt 模板
- **规则市场**：用户可分享 / 安装他人的整理规则
- **轻量协作**：共享一个项目视图（只读链接 + 评论）

### 6.5 智能升级

- **多智能体**：摘要 / 分类 / 整理 / 问答 各自独立 agent，可单独换模型
- **意图记忆**：长期记住用户偏好（"客户 A 的文件总放工作目录"）
- **预测性提示**：「你最近常打开 X，要不要把 X 锁定到侧栏？」
- **跨模态搜索**：文本搜图、图搜文本

### 6.6 v1.0 验收

- [ ] 付费转化率 ≥ 5%
- [ ] NPS ≥ 40
- [ ] 公网 GitHub Star ≥ 1k
- [ ] 至少 1 个 "killer use case"（口碑传播点）

## 7. 商业模式与定价假设

### 7.1 Freemium 三档

| 档位 | 价格 | 谁用 | 限制 |
|---|---|---|---|
| **Free** | ¥0 | 个人轻度 | 索引 < 2 万文件 / 本地模型 / 无云端 AI |
| **Pro** | ¥39/月 或 ¥299/年 | 知识工作者主力 | 无限文件 / 云端 AI ¥30/月额度 / 完整功能 |
| **Studio** | ¥79/月 或 ¥599/年 | 设计师 / 研究者 | Pro + ¥100/月 AI 额度 + 高级图谱 + 优先支持 |
| **Edu** | ¥19/月 | 学生 | Pro 功能减半额度 |

> 关键策略：**云端 AI 用量打包，不让用户算 token**。用户付的是"够用安心"，不是"运营成本"。

### 7.2 关键转化漏斗（假设）

```
下载         100%
完成扫描      80%   ← 北极星 v0.2
D1 留存       60%
D7 留存       40%   ← 北极星 v0.5
D30 留存      25%
免费 → Pro    5%    ← 北极星 v1.0
Pro 年费续费   70%
```

### 7.3 增长策略（按阶段）

| 阶段 | 增长来源 |
|---|---|
| 0-1k 用户 | 个人圈子 / 小红书 / 即刻 / X (Twitter) 自发推荐 |
| 1k-10k | 内容营销（B 站 / 公众号「我的 AI 工作流」） |
| 10k-100k | KOL 合作 / 付费投放试水 |
| 100k+ | 品牌 / 出海（英文版） |

### 7.4 不做什么

- ❌ 不做企业版 IPO 路线（个人工具基因）
- ❌ 不做"AI 助理总管"什么都管（焦点丢失）
- ❌ 不做免费版广告（毁掉「平静」原则）
- ❌ 不做硬件捆绑（与定位冲突）

## 8. 技术债与重构计划

### 8.1 短期内必还的债

| # | 问题 | 解决方案 | 工期 |
|---|---|---|---|
| 1 | 0 测试覆盖 | 至少加 Rust 后端核心逻辑单测（db / scan / ai）目标 50% | 5 天 |
| 2 | 无 CI | 加 GitHub Actions：cargo test + tsc + lint | 1 天 |
| 3 | `db.rs` 548 行单文件 | 拆成 `db/{mod,schema,queries,fts,relations}.rs` | 1 天 |
| 4 | 错误处理 String 化 | 自研 `Error` enum + `thiserror` | 1 天 |
| 5 | 前端组件粒度 | `Chat.tsx` 拆 `MessageBubble`, `ChatInput`, `Suggestions` | 1 天 |
| 6 | API key 明文存 .env | 至少存 macOS Keychain | 1 天 |

### 8.2 中期需要的架构升级

- **任务队列**：现在 Rust 启动期 seed 是同步，未来批量摘要需要持久化任务队列（建议 `tokio` + SQLite 队列）
- **状态管理**：现在前端纯 useState，未来 Settings 联动多页面需要 Zustand 或 Jotai
- **类型同步**：Rust struct → TS type 现在手抄，未来用 `ts-rs` 或 `specta` 自动生成
- **国际化**：i18next + 至少中英双语
- **可观测性**：本地结构化日志 + 用户主动反馈时打包上传

### 8.3 长期架构演进

- **多进程模型**：扫描 / 嵌入 / AI 调用各自独立子进程，crash 不影响主 UI
- **WebAssembly 内容提取器**：插件化的文件类型解析器
- **同步层**：CRDT 用于多设备同步 (Phase 4+)
- **替换 SQLite**：如果索引超 10M 文件，考虑 DuckDB 或 LMDB

## 9. 关键风险与对冲

| 风险 | 严重度 | 概率 | 对冲方案 |
|---|---|---|---|
| 智谱 API 限流 / 涨价 / 关停 | 🔴 高 | 中 | 抽象 LLM 层，支持多 provider（Claude / OpenAI / Qwen） |
| 本地 LLM 体验不够好导致免费版口碑差 | 🔴 高 | 高 | 严选模型（建议 Qwen2.5-7B-Instruct + 调优）+ 充分预期管理 |
| macOS 全盘访问授权流程劝退用户 | 🟡 中 | 高 | Onboarding 大量图文教学 + 失败兜底 + 沙盒模式 |
| 用户对"数据发云端"不信任 | 🔴 高 | 高 | 透明的审计日志 + 一键暂停 + Privacy Gateway + 文档透明 |
| 苹果 App Store 政策变化（如禁全盘扫描）| 🟡 中 | 中 | 备好直接下载渠道（DMG）作为后备 |
| 竞品（Hazel / DEVONthink / 苹果 Spotlight 升级）出 AI 版 | 🟡 中 | 高 | 加速核心差异化（图谱 + Chat），构建用户社区 |
| 团队规模小，长尾问题响应慢 | 🟢 低 | 高 | 文档先行 + 社区驱动 + 主推免费版口碑 |
| 数据丢失事故（写操作 bug） | 🔴 极高 | 低 | 所有写操作可回滚 + 自动备份 + 灰度发布 |

## 10. 需要老板拍板的决策

> 这些是路线图能否落地的关键岔路口，建议尽快对齐。

### 10.1 战略层

1. **要不要做团队 / 企业版？**
   - 选 A：纯个人 → 节奏快、决策简单
   - 选 B：兼顾团队 → 收入天花板高，但产品复杂度 ×3
   - **建议**：v1.0 之前坚决只做个人

2. **第二平台是 Windows 还是 Web？**
   - Windows：用户量大，技术栈通（Tauri 自带）
   - Web 只读：传播门槛低（分享链接），但功能受限
   - **建议**：v1.0 → Windows，v1.5 → Web 查看器

3. **要不要开源？**
   - 全开源 → 社区飞轮，但商业化困难
   - Source-available（如 Sentry 模式）→ 平衡
   - 闭源 → 商业最直接
   - **建议**：核心 Rust 引擎开源，UI + 云端服务闭源（参考 Obsidian）

### 10.2 战术层

4. **第一波内测人群优先级**？（30 个名额怎么分）
   - 建议：15 产品经理 + 10 设计师 + 5 研究者
   - 理由：PM 反馈最快，设计师视觉敏感，研究者数据密集

5. **是否做付费扫码内测？**
   - 收 1 元 → 过滤伸手党 + 心理承诺
   - **建议**：是

6. **AI 默认模型**？
   - glm-4-flash（免费但弱）
   - glm-4-air（便宜稳，推荐）
   - glm-4 / claude（贵但顶）
   - **建议**：免费版用 air，Pro 默认 glm-4，可选 claude

7. **隐私边界**：哪些数据可云端？
   - 选 A：文件名 + 摘要可云端，原文绝不
   - 选 B：用户自选目录可云端
   - **建议**：默认 A，敏感目录用户标记后只本地

## 12. 一句话总结

> **当前**：FileMind 是个能跑、能看、有 AI 的 demo，但**所有"操作型"功能都还没做**——这正是它最大价值所在。
>
> **下一步**：花 2 周把体验坑填掉，再花 2 个月把"写操作 + 实时监听 + 向量搜索"三件套做出来。然后才能谈商业化。
>
> **最大风险**：差异化锚点（**关系图谱的清理决策辅助**）必须尽快做出来。否则就是另一个被 Spotlight + ChatGPT 桌面版碾过的工具。
>
> **第一波用户**：找 30 个产品经理 + 设计师朋友，付 1 元入群，每周访谈，跑数据。
